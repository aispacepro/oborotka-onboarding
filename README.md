# Oborotka.by — Онбординг клиента

Модуль идентификации клиентов на факторинговую платформу **Oborotka.by** × **ЗАО «Нео Банк Азия»**.

Реализует полный цикл сбора данных по требованиям комплаенса банка (ПОД/ФТ/FATCA) на основе Приложения 12 — Вопросника клиента-организации, плюс согласия физлиц на проверку в государственных базах данных (БКИ НБРБ, ОАИС/ФСЗН, МВД).

## Стек

- **React 18** + **Vite 6** (быстрая сборка, HMR)
- **Tailwind CSS 3** (utility-классы)
- **lucide-react** (иконки)

## Установка и запуск

```bash
# Установить зависимости
npm install

# Запуск дев-сервера (http://localhost:5173)
npm run dev

# Production-сборка
npm run build

# Превью production-сборки
npm run preview
```

## Структура онбординга — 7 шагов

| № | Шаг              | Что собираем                                                        |
|---|------------------|---------------------------------------------------------------------|
| 1 | Начало           | УНП, email, телефон, выбор роли (кредитор/должник/оба)              |
| 2 | Анкета компании  | Реквизиты, юр. адрес, ОКЭД, контакты, санкционные списки            |
| 3 | Руководство      | Директор, главбух, уполномоченные лица                              |
| 4 | Бенефициары      | Бенефициарные владельцы + учредители (ФЛ/ЮЛ) + доп. вопросы 5.1–5.3 |
| 5 | FATCA            | Налогоплательщик США / бенефициары США / финансовое учреждение      |
| 6 | Согласия         | 4 согласия × N физлиц (ПД, БКИ, ОАИС/ФСЗН, МВД)                     |
| 7 | Подписание ЭЦП   | Сводка, документы, подтверждения, подпись                           |

## Источники данных (комплаенс)

Модуль построен на основе:

1. **Приложение 12** — Вопросник клиента-организации к Правилам внутреннего контроля ЗАО «Нео Банк Азия» в сфере ПОД/ФТ/FATCA
2. **Согласие на обработку ПД для проверки ОАИС/ФСЗН** — АИС «Паспорт» + Реестр индивидуальных лицевых счетов ФСЗН
3. **Согласие на сведения о правонарушениях** — единый государственный банк данных МВД РБ
4. **Согласие на запрос в БКИ** — Кредитный регистр НБРБ

## Интеграция с основным порталом

Этот модуль — **отдельный поток до входа в кабинет**. После успешного подписания ЭЦП:

1. Анкета и согласия отправляются в банк на скоринг
2. Аналитик банка проверяет данные (1–2 дня)
3. Скоринг и установление лимита (автоматически)
4. Уведомление клиенту на email
5. Активация — переход в основной портал (`oborotka-client-portal`)
